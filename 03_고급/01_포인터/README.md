# 09. 포인터

## 학습 목표
- 포인터 변수의 개념과 선언/역참조를 이해한다.
- `&`(주소 연산자), `*`(역참조 연산자) 사용법을 익힌다.
- 배열과 포인터의 관계를 이해한다.

## 개념 정리

### 1) 포인터 = 주소를 담는 변수
```c
int x = 42;
int *p = &x;        // p는 x의 주소를 저장
printf("%d\n", *p); // 42
*p = 100;           // x의 값이 100으로 바뀜
```

### 2) 포인터 자료형 크기
64-bit 시스템에서 모든 포인터는 보통 `8 byte`.
```c
printf("%zu\n", sizeof(int *));   // 8
```

### 3) 배열 == 포인터?
```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;          // arr은 첫 원소의 주소
printf("%d\n", p[2]);  // 30
printf("%d\n", *(p+2));// 30
```
배열의 이름은 (대부분의 식에서) 첫 원소의 **포인터로 변환**됩니다.
단, `sizeof(arr)`은 배열 전체 크기, `sizeof(p)`는 포인터 크기.

### 4) NULL 포인터
```c
int *p = NULL;
if (p != NULL) *p = 0;
```
유효하지 않은 포인터의 역참조는 **세그멘테이션 폴트**를 일으킵니다.
